## 2025-04-14 - [Redacted Hardcoded API Tokens from Exported n8n Workflow]
**Vulnerability:** The exported `n8n/whatsapp_agent_workflow.json` contained hardcoded, live API access tokens for `app.chatwoot.com` in multiple HTTP Request nodes. These tokens were exported directly from the live n8n instance and committed to the repository, thereby exposing credentials publicly.
**Learning:** Tools that export application state or workflows (like n8n JSON exports) often serialize all configured parameters—including raw secrets and API tokens. Developers frequently commit these artifacts without sanitizing them first, mistakenly treating them as configuration rather than credential stores. This represents a significant vector for secret leakage.
**Prevention:** Always implement a sanitization step before committing exported configuration files (such as `.json` workflows, Postman collections, or environment `.env` templates). Replace live secrets with safe placeholders (e.g., `[REDACTED_API_TOKEN]`) or integrate a pre-commit hook (like `trufflehog` or `git-secrets`) to automatically scan for and block the commit of known secret patterns and authentication tokens.
