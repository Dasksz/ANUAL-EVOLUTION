## 2026-04-13 - [n8n Workflow Fallback & Missing Data Configuration]
**Vulnerability:** Empty responses from the LLM caused the `Limpar Mensagem IA1` node in n8n to execute a default string fallback (`|| "Aguarde um instante..."`). This bypass triggered a false positive branch execution where users were falsely informed they were being transferred. Also, the Supabase tool for checking client registry data was missing specific column return configurations, causing the agent to hallucinate or mark missing data like 'filial' as 'N/A' while it was actually available in the db.
**Learning:** In LLM integration workflows (e.g., n8n, Make), fallback data assignments for sanitized strings shouldn't create conversational side effects if the AI genuinely outputs an empty state. Relying on default `alwaysOutputData` with poorly constructed node fallbacks leads to logical bugs. Also, when an AI agent isn't fetching the necessary columns, check the tool nodes strictly for their API `columns` parameter configuration.
**Prevention:** Remove side-effect triggering text in the string sanitizer. Explicitly define `"columns": "razaosocial, fantasia, cnpj, codigo_cliente, bairro, cidade, filial, bloqueio, ultimacompra"` and `"returnFields": "selected"` on the n8n Supabase tool node configuration to guarantee all required metadata is returned to the context. Also added strict rules to prompt to handle missing or null data by silently omitting the lines rather than hallucinating "N/A".
