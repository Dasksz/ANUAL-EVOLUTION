INTO v_var;` can sometimes be brittle with line concatenation and syntax variations in Postgres PL/pgSQL blocks. In addition, always be careful to avoid declaring variables (e.g. `v_sql text;`) multiple times inside the same `DECLARE ... BEGIN` block to prevent `ERROR: 42601: duplicate declaration` during script compilation. **Ação:** Standardized dynamic SQL string executions to use intermediate `v_sql` and ensured single declarations per function context across `full_system_v1.sql`.
