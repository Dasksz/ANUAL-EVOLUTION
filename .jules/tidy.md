## 2024-05-15 : (Fix PL/pgSQL variable evaluation context) **Aprendizado:** `EXECUTE '...' INTO v_var;` cannot simply be used as `... )' INTO v_var;` directly appended to the dynamic execution string block when parsing statements. It evaluates correctly but is much safer to set it to a local string variable (e.g., `v_sql := '...';`) and then execute it via `EXECUTE v_sql INTO v_result;` to prevent SQL injection edge cases and PostgreSQL parser ambiguity where it searches for a relation or misplaces the variable binding. **Ação:** Replaced the direct evaluation string block `)' INTO v_result;` with a two-step `v_sql := '...'; EXECUTE v_sql INTO v_result;` inside `get_dashboard_filters` in `full_system_v1.sql`.
