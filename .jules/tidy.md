## 2024-05-24 : (Padronização de Strings)
**Aprendizado:** Quando houverem mapeamentos dependentes de strings externas como nomes de municípios e cidades na aplicação (`src/js/worker.js`), evitar usar apenas `.trim().toUpperCase()` pois letras acentuadas (ex: `ILHÉUS`) causam desintegração nos relacionamentos do banco de dados (RLS/JOINs), especialmente com strings vindas de diferentes planilhas e fontes (IBGE, etc).
**Ação:** Criada utilidade `normalizeCityName(city)` baseada em `.normalize("NFD").replace(/[\u0300-\u036f]/g, "")` para normalizar globalmente os nomes de cidades, unificando os mapeamentos de configuração (`config_city_branches`).

## 2024-05-27 : (Simplificação de Renderização de Tabelas)
**Aprendizado:** A base de código continha múltiplas funções (como `renderLpTable`, `renderTable`, `renderRankingTable`) construindo linhas e células de tabelas via extensos blocos de `document.createElement('tr')`, `.createElement('td')` e `.appendChild()`. Essa verbosidade oculta a real estrutura da UI (template) e torna o código extremamente longo e difícil de manter.
**Ação:** Refatorei `renderLpTable` para usar uma única atribuição `innerHTML` combinada com `array.map().join('')` e literais de template, utilizando sempre o utilitário `escapeHtml()` para proteger contra XSS em interpolações dinâmicas. Esse padrão deve ser priorizado para construção de UI ao invés da manipulação iterativa manual do DOM, pois oferece código declarativo, limpo e performático.
## 2024-05-18 : Batching DOM creation inside custom dropdowns **Learning:** Rendering many options inside custom dropdowns using loop-based `document.createElement()` and `appendChild()` causes a lot of redundant code and can be difficult to read compared to modern template literals. **Action:** Replace verbose `createElement` loops with a batched array `.map(...).join('')` assigned to `.innerHTML` in custom dropdown initializations, but explicitly `escapeHtml()` any dynamically sourced values (like option text/values) to prevent XSS. Event listeners must be queried and attached post-render.
