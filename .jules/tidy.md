## 2024-05-13 - Correção Faturamento Consolidado **Aprendizado:** PostgREST relies on exact match of data types between JSON payload and SQL function signatures. If the filter is derived from an object containing numbers (e.g. `filters.p_ano`), passing it as a number will cause `PGRST202: no matches were found in the schema cache` error when the RPC expects a text argument. **Ação:** Sempre converta explicitamente tipos não-nulos para os tipos esperados pela RPC usando construtores como `String()` ou `parseInt()` ao construir o payload JSON no lado do cliente.
## 2024-05-16 : Padronização de Dropdowns de Data **Aprendizado:** A renderização repetitiva de elementos select HTML (`anoSelect` e `mesSelect`) usando `.map(...).join('')` polui vários módulos. A duplicação dessa lógica (incluindo tratamento de valores padrão como 'Todos') é propensa a erros. **Ação:** Extrair ou sempre utilizar as funções utilitárias do `utils.js` (como `generateYearOptionsHtml` e `generateMonthOptionsHtml`) ao criar novos dropdowns que envolvam dimensões temporais, centralizando a lógica de sanitização XSS e os defaults da aplicação.
