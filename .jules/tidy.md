## 2024-06-27 - [Fixing SQL Order Pricing and Filtering]
**Aprendizado:** When a table blends multiple transaction types (e.g. sales vs. bonuses), blind COALESCE statements may favor the wrong column if one defaults to 0. vlbonific must explicitly be used for bonus order types (like '5' and '11') to ensure they don't display as zero when queried.
**Ação:** Updated sp_consultar_pedido and sp_historico_pedidos to conditionally select vlbonific or vlvenda based on tipovenda. Also added instructions for the N8N agent to loop through Mestre_Busca_PRIME for specific multi-order queries like 'which of these have Kerococo?'
