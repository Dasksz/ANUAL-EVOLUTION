## 2026-05-18 - Dynamic ARIA Labels and Pressed/Expanded States on UI Toggles
**Learning:** Icon-only buttons used for toggling states (like dropdown menus or settings panels) can become confusing for screen reader users and fail accessibility requirements when missing dynamic ARIA states. We fixed custom dropdown toggles (like export dropdown and profile dropdown) that lacked `aria-expanded` syncing and click-away management.
**Action:** When implementing dropdowns or state-toggling buttons, always initialize them with `aria-expanded="false"`. In the event listener that toggles the state, dynamically update `aria-expanded` to reflect the current state. Update the clickaway utility handlers to toggle `aria-expanded` to false when clicked away. Ensure `focus-visible` ring classes are applied for keyboard accessibility.
