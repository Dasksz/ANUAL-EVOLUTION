## 2024-05-18 - Avoid refactoring DOM innerHTML templates and Event Delegation.
**Learning:** For rendering complex custom dropdown lists, using \`innerHTML\` and a single Event Delegation listener on the parent container provides significant O(N) performance improvements over iteratively using \`document.createElement\` and binding individual event listeners per child element.
**Action:** When implementing custom dropdowns (like `setupMultiSelect`), write the HTML template as a large string using mapped arrays and `.join('')`, assign it to \`innerHTML\`, and attach a single `click` or `change` listener to the parent element. Always sanitize user data via `escapeHtml()`.
