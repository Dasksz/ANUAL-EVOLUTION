## 2025-02-18 - Batch clear_summary_month operations
**Learning:** Sequential network requests in nested loops (`await clear_summary_month` inside year/month loops) create significant throughput bottlenecks. Batching lightweight RPCs with `Promise.all` can drastically reduce overall time spent waiting on network latency without causing database strain.
**Action:** Extracted the `clear_summary_month` operations out of the granular execution loops and grouped them into an array of promises, executing them concurrently with `Promise.all` before beginning the sequential processing loop. This resulted in a >95% improvement in processing latency for the clearance phase.
