## 2026-06-19 - Redução drástica de contexto de Agentes LLM usando Banco de Dados
**Learning:** Forçar um Agente Langchain a realizar regras de negócio ("se valor < 1500, recomende X") e cruzar dezenas de tabelas json recebidas por Tool Calls infla absurdamente o uso de Tokens (podendo bater 40k+ em meia dúzia de turnos) e esgota limites de raciocínio lógico (ex: erro `reasoning_content` em Modelos DeepSeek).
**Action:** Construir arquiteturas onde a inteligência mora em `Materialized Views` e `RPCs` no banco (SQL é infinitamente mais barato e rápido para cruzar tabelas). O Agente V1 foi dividido em: LLM Roteador que só entende a intenção e cospe JSON -> Nó Switch (n8n) -> Chamada Direta de RPC -> LLM Simples só para formatar texto do WhatsApp.
